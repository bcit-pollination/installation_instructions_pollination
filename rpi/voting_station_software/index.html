Voting Station Software installation instructions

Overview#

In this tutorial you will:

  • Install MongoDB
  • Install NodeJS
  • Prepare your RPI voting station to use bluetooth
  • Install the RPI voting station software
  • Understand how to use the RPI voting station software

Requirements#

To successfully complete this tutorial you will need:

  • a USB with enough space and a name without spaces (ex: "USB", not "U SB"),
  • a Raspberry Pi with:
    • Raspbian OS
    • Bluetooth
    • Node js version 8.9.4
    • Mongo version 2.4.14
    • Python 3
  • a touchscreen (optional).

Procedures#

Install MongoDB#

  1. Update your machine
sudo apt update
sudo apt upgrade
  1. Begin the installation
sudo apt install mongodb
  1. Check the installed MongoDB version
mongo --version
Note

Make sure the version is 2.4.14.

  1. Navigate to the root folder of your system.
cd /
  1. Create a data/db/ directory.
sudo mkdir data
sudo mkdir data/db/
  1. Start MongoDB service.
sudo service mongodb start
  1. Check for MongoDB connections.
sudo mongod
  1. Confirm a mongod.lock file was created inside data/db/.
cd data/db/
ls

Install Node JS#

  1. To download and install newest version of Node.js, use the following command
curl -sL https://deb.nodesource.com/setup_8.x | sudo -E bash -
  1. Begin the installation
sudo apt-get install -y nodejs
  1. Check the installed Node.js version
node -v
Note

Make sure the version is 8.9.4.

Prepare RPI voting station to use bluetooth#

Enter the following commands to set up your Raspberry Pi's Bluetooth

sudo service bluetooth stop
sudo update-rc.d bluetooth remove
sudo systemctl stop bluetooth
sudo systemctl disable bluetooth
sudo apt-get install bluetooth bluez libbluetooth-dev libudev-dev
Important

At this stage, please restart your RPI

Important

Once restarted, make sure that your Bluetooth is turned on.

Install the RPI voting station software#

  1. Choose a directory of your choosing and clone the project at https://github.com/bcit-pollination/voting_station_pollination.git onto your Raspbian
git clone https://github.com/bcit-pollination/voting_station_pollination.git
  1. Navigate to the root directory of the project
cd voting_station_pollination/
  1. Install the required dependencies
sudo npm install -g electron --unsafe-perm=true --allow-root
npm install
pip install cryptography
  1. Check your mongoose version.
npm list | grep "mongoose"
Note

Make sure the version is 2.9.10.

How to use the RPI voting station software#

Running the graphical interface#

  1. From the root directory, navigate to the touchscreen_electron folder
cd touchscreen_electron
  1. Run the program
npm start

Run Control Server#

  1. Click on "Run Control Server".

  2. Log in with the credentials and click "Login".

  3. After you are done logging in, click on ‚ÄúGet Election List‚Äù.

    The application will generate the current Election List that the logged in organization has.

  4. Select an election of your choosing, and click on "Download Election Package".

    The "Usb List" section will be displayed.

  5. Plug your USB into the Raspberry Pi.

    Important

    Make sure you USB's name does not have space. (ex: "USB", not "U SB")

  6. Click on "Refresh" to have your USB path showed in the application.

  7. Click on the USB path of your choosing to have the Election Data exported to.

  8. Click on "Export Election Data".

  9. At this moment, you will see the Encoded Election Data exported to the root of your USB

Note

With every new Election Package you exported from an election, new Encoded Data will overwrite the old Encoded Data

  1. Open another terminal and enter the MongoDB shell to switch to the Pollination database.
mongo
use pollination
  1. Go back to the Navigation Page.

Run Voting Server#

  1. Click on "Run Voting Server".

  2. Log in with the same credentials and click "Login(as Voting Pi)".

  3. Click on "1. Identify USB" to have your USB path showed in the application.

  4. Choose your USB with the exported Election Data saved on it.

  5. Click on "2. Import Election Data".

    An alert will appear saying that the Election Package was successfully imported, press OK.

  6. Enter your Verifier Password to start the BLE Server, and press "Submit Verifier Password".

    An alert will appear saying that the verification was successful, press OK.

  7. Enter the polling station's location, and press "Submit".

  8. Click "Start BLE Server" to start the voting server.

    At this moment, touchscreen voter is allowed to enter their voting token to be granted with the permission to vote by the Verifier.

Now, users will be able to vote on the Raspberry Pi touchscreen or connect to the Raspberry Pi on their mobile devices via Bluetooth.

tip

For instructions on installing the iOS app: iOS Instructions

For instructions on installing the Android app: Android Instructions

Voting on Touchscreen#

  1. Once the touchscreen voter has finished with their voting, press "Submit Vote".

  2. The Verifier then click "End Vote" to export the vote data

  3. The Verifier is then required to enter their password in order to proceed to exporting. Once entered, press "Submit Verifier Password".

    The USB list will then be visible.

  4. Click on "show USB Devices" and select the USB you want to export the vote data to

  5. Click on "Export Vote Data".

    At this moment, you can then proceed onto posting the Election Results.

How to Post the Election Results#

  1. Click on "Run Control Server", and log in with the credentials.

    ![Run Control Server]

  2. Click on "Post Election Results"

    The USB List section will be displayed.

  3. Click on "Refresh" and select the USB you want to import the vote results from

  4. Click on "Import Vote Results".

    An alert will appear saying that the import was successful. After pressing OK, the "Upload Election Results" button will be visible.

  5. Click on "Upload Election Results"

    An alert will appear saying that the upload was successful, then you are done.

Troubleshooting#

Incorrect Node JS version#

If required Upgrading or Downgrading, please follow the steps inside https://www.surrealcms.com/blog/how-to-upgrade-or-downgrade-nodejs-using-npm.html to do so
